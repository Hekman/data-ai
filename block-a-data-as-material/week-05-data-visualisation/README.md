# Block A · Week 5 — Data Visualisation

> **Block question:** *What does this data actually let me know, and what stories can I tell?*
> **This week's question:** *How do I choose the visualization that answers a question — and edit it down until the insight is unmissable?*

---

This week is about matching chart type to question and applying the encoding principles that separate a clear chart from a decorative one. You build static charts in Seaborn and Matplotlib and treat each one as an argument: what to foreground, what to strip away, how to arrange what remains so the reader can reason. Honesty is a constraint here, not an afterthought — truncated axes and hidden subgroups mislead even when the arithmetic is correct. (Next week you make these views interactive and wrap them into a dashboard.)

**Why this matters for designers**

This is the most visibly 'designerly' week, and the temptation to over-design is strongest. A chart is an argument. Good data visualisation is an act of editing: removing everything that doesn't help the reader reason, and arranging what remains so the insight is unmissable. Sequence, framing, and emphasis matter as much as the chart itself.

The slides for this class will be available here after class.

**By the end of this week you can:**

- Match chart type to question and apply core encoding principles correctly.
- Build and customise static charts (Seaborn/Matplotlib) for a defined audience.
- Spot and avoid misleading encodings (truncated axes, hidden subgroups, chart junk).
- Sequence a few charts into a short, honest narrative.
